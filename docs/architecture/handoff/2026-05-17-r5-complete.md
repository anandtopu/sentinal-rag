### R3.S6 — Embedder hoist

Per-request `LiteLLMEmbedder(...)` construction in `Orchestrator.run`
replaced with a process-singleton hoisted onto `app.state.embedder` in
`apps/api/app/lifecycle.py`. Wired through `apps/api/app/dependencies.py`
(`EmbedderDep`) and `apps/api/app/api/v1/routes/query.py` (passed into
`Orchestrator(..., embedder=...)`).

The Temporal worker keeps the per-request construction path — its
embedding-model alias varies per evaluation run via
`model_config["embedding_model"]`, so the Orchestrator constructor
keeps both `embedding_model` + `ollama_base_url` kwargs (the worker's
contract) and adds an optional `embedder` override (the API's hot
path). When `embedder` is provided, the existing fields are unused
for construction; when it's None, the legacy per-request path runs.

## Decisions made during R5

The six ADRs are themselves the decisions. Key trade-offs picked
without prior locking:

1. **R5.S2 (migrations) — class taxonomy + CI gate.** The plan asked
   for "expand → backfill → contract." We added a four-class
   taxonomy (A/B/B'/C) so review-time classification is unambiguous,
   and a CI gate that classifies every revision file. Class C
   (type rewrites, in-place renames) is forbidden at merge — they
   must be reshaped as expand→contract.
2. **R5.S3 (injection) — no ML input scanner in v1.** The plan
   said "in-context defense only is a defensible position." We
   committed to that explicitly and documented the upgrade path
   (small classifier co-located with the retrieval-service NLI pod,
   once a real customer demands it).
3. **R5.S4 (PII) — Presidio over scrubadub / commercial services.**
   Presidio's `PERSON` NER is the differentiator; commercial services
   contradict the self-hosted posture from ADR-0014. Per-tenant
   opt-in via Unleash so tenants with technical docs (security,
   training material) don't pay the false-positive cost.
4. **R5.S5 (sharding) — partial per-tenant HNSW, not per-tenant
   tables.** Keeps the RLS + single-schema property intact. The
   carve-out is operator-initiated; the detection job warns but
   does not act. Documented exact triggers (5M chunks, 25% share,
   or recall@10 < 0.85).
5. **R5.S6 (streaming) — new endpoint, not a `?stream=true` flag.**
   Different response media type (`text/event-stream` vs
   `application/json`); different idempotency story; mixing them
   under one path breaks OpenAPI schema fidelity.

## Risks the next session should pay attention to

1. **None of the ADRs' implementations have shipped.** ADRs document
   intent; the code follows in separate PRs. A reviewer reading
   "ADR-0032 says we do RTBF" must understand that the workflow,
   migrations, and route do NOT exist in the repo yet. The
   "Implementation work these ADRs unlock" table above is the
   authoritative scope.
2. **The Embedder hoist is the API path only.** The Temporal
   worker still constructs its own per-call embedder by design.
   Anyone debugging "the worker doesn't use the hoisted embedder"
   should re-read the trade-off in `orchestrator.py`'s docstring +
   this handoff — the worker's per-call model override is the
   reason.
3. **`embedder.loaded` logged at startup will fail loud** if the
   `default_embedding_model` is invalid (not in `EMBEDDER_DIMENSIONS`).
   That's intentional — better to fail at boot than the first
   `/query`. Operators changing the embedding model in `.env`
   without updating the dimensions table will see the failure
   immediately.
